## 1. How to read this document

### Status legend

| Status | Meaning |
|---|---|
| `TRUE` | Verified working. Evidence exists and is cited. |
| `PARTIAL` | Implemented and runs, but has a known defect or is incomplete. |
| `FALSE` | Not implemented, or known to be fully broken. |
| `UNKNOWN` | Not assessed. No evidence either way. **Honest default — never guessed as TRUE.** |

### Priority legend

| Priority | Meaning |
|---|---|
| `P0` | **Launch blocker.** Editor trust. If not TRUE, Rwanga cannot launch. |
| `P1` | Required for a credible launch. Not an absolute blocker, but expected. |
| `P2` | Should follow launch closely. Not blocking. |
| `P3` | Future / v2+ / "if planned". Not blocking. |

### Honesty rules for filling this table

1. **No `TRUE` without cited evidence.** A passing unit test, a commit SHA, a diagnostic report, or a recorded manual-QA pass.
2. **`UNKNOWN` is marked honestly.** If it has not been verified, it is `UNKNOWN` — not optimistically `TRUE` or `PARTIAL`.
3. **Code existing is not evidence of working.** The "Evidence required" column states exactly what proof would flip the item to `TRUE`.

## 2. Operating rules (binding)

1. **Rwanga cannot launch while any P0 is FALSE, PARTIAL, or UNKNOWN.** (Rule 0, repeated because it is the whole point.)
2. **Do not start invention features until P0 editor trust is TRUE.** No new capabilities, no "wouldn't it be nice", no scope additions while a single P0 is open.
3. **New ideas go to Stage 3 only — and Stage 3 opens only after foundation lock.** Ideas raised before foundation lock are recorded, not built.
4. **No `TRUE` without evidence.** See §1.
5. **Section 2 (Manuscript truth) and Section 3 (RTL) are one coupled campaign.** They share the same pagination engine. The 2026-05-20 recovery failed because it tried to slice them apart. Do not slice them apart again.
6. **Every step updates this checklist — no exceptions.** No repair, fix, or verification step is complete until this document is updated *in the same step*. The update must state: (a) every checklist **ID** whose **status changed** and its new status; (b) the **evidence added** for any item moved toward `TRUE` (test name, commit SHA, diagnostic report, or recorded QA pass); (c) the **still-blocking** items that remain; (d) the exact **checklist IDs affected**. A step that does not update the checklist did not happen.
7. **"Launch" and "finished" are two different bars.** Rwanga **cannot launch** while any **P0** is `FALSE`, `PARTIAL`, or `UNKNOWN`. Rwanga **must not be called finished** until **every item (P0–P3, all 233)** is `TRUE` with cited evidence. Neither bar may be declared met by assertion — only by evidence in this document.
8. **Geometry ownership invariant — page leaves are content-range-bound, never decorative.** PageSurface may render a page leaf **only if** its boundaries are sourced directly from a PageMap page range. Each leaf corresponds **1:1 to `PageMap.pages[i]`** and owns that page's **start block, end block, page number, and fixed geometry**. Forbidden: decorative page leaves, fake page shells, visual pages independent of content ranges, page count inferred from CSS or DOM. PageMap page count changes ⇒ leaf count changes; PageMap page boundaries change ⇒ leaf boundaries change. **No visual page structure may exist independent of PageMap.** Bears on MT-01, MT-09, MT-11 — a leaf that fails this invariant cannot be cited as evidence for any of them.
9. **Hidden editor is state-preservation only — never a second geometry owner.** While Paper view is active, the hidden Flow editor preserves **only** selection, undo history, and plugin state. It is **forbidden** to: query the hidden editor for geometry, page count, or page boundaries; use it as rendering truth; or read `getBoundingClientRect()` / `scrollHeight` / `clientHeight` for any geometry purpose. Paper truth flows one way only — **RenderModel → PrintRenderer → Paper surface.** The hidden editor is state, not truth.
10. **Manuscript truth is the ratified Kurdish/RTL screenplay profile — not Word, not a page-count target (Option C, ratified 2026-05-21).** Rwanga paginates to a **Kurdish/RTL screenplay profile**: screenplay *structure* taken from standard screenplay convention; *typography* (font, cpi, per-type column widths, margins, line-height, paragraph air) chosen deliberately for Kurdish RTL readability. **The page count is an output of that profile, never a target to hit.** The synthetic "Word ≈ 40 pages" figure is **retired as a launch metric** — Density Slice 5 proved it a non-reproducible reconstruction of a generic, non-screenplay DOCX; it survives only as a historical, non-reproducible reference. For every Section-2 (Manuscript) and Section-3 (RTL) item, the `TRUE` bar is: **PageMap, Paper view, Print Preview, and (later) PDF export all agree with the ratified Kurdish/RTL profile.**
