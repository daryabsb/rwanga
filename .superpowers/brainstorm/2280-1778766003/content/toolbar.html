A screenplay is mostly plain Courier. Bold/italic are rare; block types (action/character/dialogue) are handled by Tab/Enter, not buttons; annotate/tag/flag already live on right-click. So the question is honest: how much persistent chrome does this page actually need? You've valued a clean page and "not invading" — here's how that applies to the toolbar.

A slim persistent strip with only context controls (current block type, undo/redo). All formatting + writer actions move to the floating selection bar.

C. Hybrid — slim context strip + floating bar

A minimal always-on strip for the two things that benefit from being persistent (block type, undo/redo), with formatting on the floating bar. Compromise between discoverability and a clean page.

My recommendation: B — floating selection toolbar only. It's the toolbar equivalent of "don't invade" — no permanent chrome, the page is the page, tools appear only when you've selected something to act on. The status bar already carries block type and active scene, so the persistent strip in A/C is mostly redundant. If you find yourself wanting block type always visible while writing, C is the fallback. Pick one — and that's the last interaction question; after this I'll lay out the full design for your approval.
