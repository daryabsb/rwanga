# Export Branding Design Task — Rwanga (ڕوانگە)

> **For Claude Design agent.** Create production-ready branded export templates for Rwanga's PDF and HTML exports. These are the documents that directors, crew, and external reviewers will receive — they represent the platform's professional identity.

## Context

Rwanga is a Kurdish cinema preproduction platform. Exports are the deliverables that leave the platform — call sheets sent via WhatsApp, scene viewers shared with crew on set (offline), shot lists printed for meetings. They must look professional, work in RTL (Kurdish Sorani), and carry Rwanga's visual identity.

## Brand Identity

- **Product name:** ڕوانگە (Rwanga) — means "perspective" in Kurdish
- **Tagline (Kurdish):** پلاتفۆرمی پێشبەرهەمهێنانی سینەمای کوردی
- **Tagline (English):** Kurdish Cinema Preproduction Platform
- **Primary brand color:** `#F72585` (Rwanga pink)
- **Accent color:** `#D4A574` (amber — used for shot numbers, scene markers)
- **Dark background:** `#0F0F12`
- **Surface:** `#17171C`
- **Text primary:** `#EDEAD8`
- **Text secondary:** `#78788C`
- **Border:** `#2C2C38`
- **Font stack:** Cairo (Kurdish/Arabic) + Inter (Latin/numeric)
- **Border radius:** 2px max
- **Shadows:** none
- **Direction:** RTL primary, LTR for technical terms and numbers

## Deliverables Required

### 1. Call Sheet PDF Template (`call_sheet_template.html`)

**Format:** HTML template rendered to PDF via WeasyPrint
**Size:** A4, portrait
**Distribution:** WhatsApp (must look good on phone screens)

**Content structure:**
```
┌─────────────────────────────────────────────┐
│  HEADER (Rwanga letterhead)                 │
│  ───────────────────────────────────────── │
│  Logo (pink ڕ mark) │ Project Title         │
│  Director │ Date │ Day Number               │
│  General Call Time (large, bold)            │
├─────────────────────────────────────────────┤
│  LOCATION                                   │
│  Name │ Address │ Map link (QR code)        │
│  Weather: {{ weather_data }}                │
├─────────────────────────────────────────────┤
│  SCHEDULE                                   │
│  Table: Time │ Scene │ Description │ Cast   │
│  (each ScheduleBlock as a row)              │
├─────────────────────────────────────────────┤
│  CAST                                       │
│  Table: Character │ Actor │ Call Time │ Notes│
├─────────────────────────────────────────────┤
│  CREW NOTES                                 │
│  Department-specific notes if any           │
├─────────────────────────────────────────────┤
│  FOOTER                                     │
│  "Generated by ڕوانگە" │ Page X of Y        │
│  Project contact info                       │
└─────────────────────────────────────────────┘
```

**Requirements:**
- Must use `@page` CSS for print margins
- WeasyPrint compatible (no flexbox — use tables/floats/grid)
- RTL layout with `dir="rtl"` on body
- Technical terms (lens mm, time codes) in LTR `<bdi>` spans
- QR code slot for location map link
- General call time must be the most visually prominent element
- Phone-readable: minimum 11pt body text, high contrast

**Django context variables:**
```python
{
    "project": Project,
    "shoot_day": ShootDay,
    "call_sheet": CallSheet,
    "schedule_blocks": QuerySet[ScheduleBlock],
    "cast_calls": [{"character": str, "actor": str, "call_time": time, "notes": str}],
    "location": Location,
    "weather": dict,
    "crew_notes": [{"department": str, "note": str}],
}
```

- Print-friendly: `@media print` styles included
- Collapsible sections via vanilla JS (no HTMX, no frameworks)
- RTL layout
- Floor plan SVG rendered from JSON data inline

## Design Tokens to Use

Use these CSS custom properties (they match `rwanga.css`):

```css
:root {
    --rw-bg: #0F0F12;
    --rw-surface: #17171C;
    --rw-border: #2C2C38;
    --rw-text: #EDEAD8;
    --rw-text-2: #78788C;
    --rw-amber: #D4A574;
    --rw-pink: #F72585;
    --rw-radius: 2px;
```

```css
@media print, .rw-print {
    --rw-bg: #FFFFFF;
    --rw-surface: #F7F7FA;
    --rw-border: #DCDCE8;
    --rw-text: #0F0F12;
    --rw-text-2: #5C5C70;

```

## Letterhead / Logo Mark

For now, use a text-based logo mark until proper branding assets exist:
```html
<span class="rw-logo-mark" style="color: var(--rw-pink); font-family: Cairo, sans-serif; font-weight: 700; font-size: 1.5em;">ڕ</span>
<span class="rw-logo-text" style="font-family: Cairo, sans-serif; font-weight: 600;">ڕوانگە</span>
```

The pink ڕ is the brand mark. Simple, recognizable, works at any size.

## What NOT to Do

- Do NOT use flexbox (WeasyPrint doesn't support it well) — use CSS Grid or tables
- Do NOT link external resources (CDN, Google Fonts, etc.)
- Do NOT use JavaScript in PDF templates (WeasyPrint ignores it)
- Do NOT use `left`/`right` CSS — use `inline-start`/`inline-end` logical properties
- Do NOT auto-translate any Kurdish text — use Django template tags
